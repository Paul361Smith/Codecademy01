Semantic HTML

Semantic HTML uses tags which have meaning related to the structure and type of content. For example, elements like <article> or <footer> explain how the content relates to the rest of the page, whereas non-semantic tags like <div> create a grouping but give no informaton about the content or how it fits in with the structure around it.

Using semantic HTML tags improves the readability of the code, as well as increasing accessibility and search engine optimisation.

Tag Name Type Usage
<header> Header Section The header typically contains a heading, navigational aids such as a list of contents, and perhaps a search box and any logo.
<nav> Nav Section Navigation links to other pages or to sections within this page
<main> Main Grouping Content Groups together the dominant contents of the document; there should only be one non-hidden main element
<footer> Footer Section The footer typically contains information about who wrote the content, links to related material, and copyright information
<section> Section Section A thematically grouped section, for example a chapter, typically with a heading
<article> Article Section A complete or self-contained composition such as a blog or comment which could be independently re-used
<aside> Aside Section Tangentially related material e.g. pull quotes or sidebars
<figure> Figure Grouping Content Typically groups together an image, diagram or photo with related caption/annotations
<figcaption> Figcaption Grouping Content Caption or legend for the content in the parent <figure> element
<video> Video Embedded Content Typically used for playing videos or movies
<embed> Embed Embedded Content For integration of external (usually non-HTML) interactive content e.g. GIFs
<audio> Audio Embedded Content A sound or audio stream
